# Design PRD: 回测引擎页面 (BacktestEngine)

> **版本**: 1.0
> **日期**: 2026-05-03
> **状态**: Draft
> **相关页面**: `/backtest`

---

## 1. 概述

### 1.1 目标
为用户提供一个专业、直观的量化策略回测界面，支持同步/异步回测、结果可视化、历史记录管理。

### 1.2 用户故事
- 作为量化研究员，我想快速配置回测参数并运行回测，以便验证策略有效性
- 作为交易员，我想查看历史回测记录的权益曲线和交易明细，以便分析策略表现
- 作为新手用户，我想通过简单的表单操作就能完成回测，无需理解底层复杂度

### 1.3 成功标准
- 首次回测配置到结果展示 < 3 次点击
- 同步回测响应时间 < 5 秒（单只股票，1年数据）
- 结果可视化加载时间 < 2 秒

## 2. 信息架构

### 2.1 页面地图

```
BacktestEngine 页面
├── 参数配置区 (BacktestForm)
│   ├── 策略选择
│   ├── 股票池输入
│   ├── 日期范围
│   ├── 初始资金
│   └── 费率设置
├── 模式切换 (ModeToggle)
│   ├── 同步模式
│   └── 异步模式
├── 进度展示 (BacktestProgress) — 仅异步模式
├── 结果展示区 (ResultsSection)
│   ├── 指标卡片 (MetricsCards)
│   ├── 权益曲线图 (EquityChart)
│   ├── 交易明细表 (TradeTable)
│   └── 详细指标 (DetailMetrics)
├── 空状态提示
├── 错误提示
└── 历史记录 (BacktestHistory)
```

### 2.2 路由与状态

| 路由 | 说明 |
|------|------|
| `/backtest` | 默认进入，显示空状态 |
| `/backtest?id={reportId}` | 加载指定回测报告 |
| `/backtest?strategy={name}&stock={code}&start={date}&end={date}` | 从 Dashboard 快捷跳转 |

### 2.3 数据模型

```typescript
interface BacktestRequest {
  strategy: string           // 策略名称
  stock_pool: string[]       // 股票代码列表
  start_date: string         // YYYY-MM-DD
  end_date: string           // YYYY-MM-DD
  initial_capital?: number   // 默认 1,000,000
  commission_rate?: number   // 默认 0.0003
  slippage_rate?: number     // 默认 0.0001
}

interface BacktestResult {
  id: string
  status: 'completed' | 'failed'
  strategy: string
  stock_pool: string[]
  start_date: string
  end_date: string
  total_return: number       // 总收益率
  annual_return: number      // 年化收益率
  sharpe_ratio: number       // 夏普比率
  max_drawdown: number       // 最大回撤
  calmar_ratio: number       // Calmar 比率
  win_rate: number           // 胜率
  total_trades: number       // 总交易次数
  portfolio_values: PortfolioPoint[]
  trades: Trade[]
}
```

## 3. 交互设计

### 3.1 用户流程

```
[进入页面]
    ↓
[填写表单 / 从 URL 加载参数]
    ↓
[选择模式: 同步/异步]
    ↓
[点击"运行回测"]
    ↓
[同步: 等待结果 / 异步: 显示进度]
    ↓
[展示结果: 指标 + 图表 + 交易记录]
    ↓
[可选: 保存到历史 / 查看历史记录]
```

### 3.2 状态机

```
                    ┌─────────────┐
                    │    Idle     │
                    └──────┬──────┘
                           │ 填写表单
                           ▼
                    ┌─────────────┐
           ┌───────│   Ready     │◄────────┐
           │       └──────┬──────┘         │
           │              │ 点击运行        │
           │              ▼                │
           │    ┌─────────────────┐        │
           │    │  Sync Running   │        │
           │    └────────┬────────┘        │
           │             │ 完成             │
           │             ▼                 │
           │    ┌─────────────────┐        │
           └───►│ Result Display  │────────┘
                └─────────────────┘
                           ▲
                           │ 异步完成
                           │
                ┌─────────────────┐
                │ Async Running   │
                └─────────────────┘
```

### 3.3 边界状态

| 状态 | 触发条件 | UI 表现 |
|------|---------|---------|
| 空状态 | 首次进入，无历史记录 | 显示提示"配置参数后运行回测" |
| 加载中 | 同步回测请求中 | 按钮 loading，禁止重复提交 |
| 异步进度 | 异步回测运行中 | 显示进度条，可取消 |
| 结果展示 | 回测成功完成 | 指标卡片 + 图表 + 交易表 |
| 错误状态 | 回测失败或加载报告失败 | Alert 提示，可重试 |
| 历史为空 | 无历史记录 | 历史区域显示空提示 |

## 7. 技术备注

### 7.1 响应式
- 桌面端 (>1024px): 4 列表单，横向指标卡片
- 平板端 (768-1024px): 2 列表单，换行指标卡片
- 移动端 (<768px): 1 列表单，垂直堆叠

### 7.2 性能
- 使用 `shallowRef` 存储 `BacktestResult` 大对象
- 图表在 `nextTick` 后渲染，避免 DOM 未就绪
- 历史记录使用虚拟滚动（>50 条时）

### 7.3 可访问性
- 所有表单元素关联 label
- 图表提供文字摘要（供屏幕阅读器）
- 颜色不作为唯一信息传递方式（收益同时显示 +/- 符号）
