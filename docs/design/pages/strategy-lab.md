# StrategyLab 策略实验室页面设计文档

> **页面路径**: `/strategy-lab`  
> **组件文件**: `pages/StrategyLab.vue`  
> **状态**: Active

---

## 1. 概述

### 1.1 目标
策略管理与参数优化中心，支持策略 CRUD、参数调优、性能对比。

### 1.2 用户故事
- 作为策略开发者，我想管理我的策略库（增删改查）
- 作为研究员，我想对策略进行参数优化（网格搜索/遗传算法）
- 作为交易员，我想对比多个策略的历史表现

---

## 4. 组件规范

### 4.1 策略卡片

**布局**: CSS Grid, `repeat(auto-fill, minmax(240px, 1fr))`

**卡片内容**:
- 顶部: 策略名称 + 类型标签
- 中部: 描述 + 最近表现
- 底部: 操作按钮（编辑/删除/回测）

### 4.2 参数优化表单

**字段**:
| 字段 | 组件 | 说明 |
|------|------|------|
| 选择策略 | NSelect | 从策略库选择 |
| 参数范围 | NDynamicInput | 参数名 + 最小/最大/步长 |
| 优化算法 | NSelect | 网格搜索 / 遗传算法 / 贝叶斯优化 |
| 目标函数 | NSelect | 夏普比率 / 收益回撤比 / 总收益 |

## 5. 交互设计

### 5.1 用户流程

```
[进入页面]
    ↓
[展示策略列表]
    ↓
[用户操作]
    ├── 创建策略 → 弹窗表单
    ├── 编辑策略 → 抽屉编辑
    ├── 删除策略 → 确认弹窗
    └── 参数优化 → 优化配置区
```

### 5.2 边界状态

| 状态 | 触发条件 | UI 表现 |
|------|---------|---------|
| 无策略 | 策略库为空 | 显示空状态 + 创建引导 |
| 优化中 | 参数优化运行中 | 进度条 + 可取消 |
| 优化完成 | 找到最优参数 | 结果表格 + 应用按钮 |

## 7. API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/strategies` | GET | 获取策略列表 |
| `/api/strategies` | POST | 创建策略 |
| `/api/strategies/:id` | PUT | 更新策略 |
| `/api/strategies/:id` | DELETE | 删除策略 |
| `/api/strategies/:id/optimize` | POST | 参数优化 |

---

## 8. 性能优化

- 策略列表分页加载（每页 20 条）
- 优化过程使用 WebSocket 推送进度
- 结果对比使用虚拟表格
