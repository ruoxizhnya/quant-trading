# 设计原则

> Quant Lab 的设计哲学与决策准则

---

## 1. 核心原则

### 1.1 数据优先 (Data First)

金融数据是产品的核心。所有设计决策以**数据可读性**为最高优先级。

- **数字使用等宽字体**：`JetBrains Mono` / `Fira Code`，确保对齐
- **精度一致**：收益率保留 2 位小数，价格保留 2 位，数量保留 0 位
- **千分位分隔**：大额数字使用 `¥1,234,567.89` 格式
- **色彩编码**：正收益绿色 `#3fb950`，负收益红色 `#f85149`

### 1.2 状态明确 (Clear States)

用户必须一眼识别当前系统状态和操作结果。

- **收益方向**：同时显示符号 (`+` / `-`) 和颜色，不依赖颜色 alone
- **加载状态**：所有异步操作显示进度或 skeleton
- **空状态**：明确提示下一步操作，不显示空白
- **错误状态**：友好提示 + 重试入口

### 1.3 暗色优先 (Dark First)

量化交易员长时间盯盘，暗色主题降低视觉疲劳。

- 背景色 `#0d1117`，文字色 `#e6edf3`，对比度 ≥ 7:1
- 避免纯白色，使用 `#e6edf3` 作为最亮文字
- 图表使用暗色主题，网格线 `#21262d`

### 1.4 响应式 (Responsive)

从 4K 显示器到笔记本都能良好展示。

- 断点：
  - Desktop: > 1200px
  - Tablet: 768px - 1200px
  - Mobile: < 768px
- 使用 CSS Grid 和 Flexbox，避免固定宽度

### 1.5 即时反馈 (Instant Feedback)

操作后 100ms 内给出视觉反馈。

- 按钮 hover: `transform: translateY(-1px)` + 边框高亮
- 点击: 涟漪效果或颜色加深
- 表单提交: loading 状态 + 禁用重复提交

## 2. 设计决策记录

### 为什么使用 Naive UI？

| 备选方案 | 不选原因 | Naive UI 优势 |
|---------|---------|--------------|
| Element Plus | 暗色主题不够深入 | Naive 暗色主题开箱即用 |
| Ant Design Vue | 组件体积大 | Naive 按需引入，体积小 |
| 自研组件库 | 维护成本高 | Naive 覆盖 90% 需求 |

### 为什么使用 CSS 变量而非 Sass？

- 运行时切换主题（未来支持亮色模式）
- 无需构建步骤，调试更直观
- 与 Naive UI 的 `n-config-provider` 配合良好

### 为什么指标卡片使用 5 列网格？

- 回测核心指标共 5 个：收益、年化、夏普、回撤、Calmar
- 5 列在 1200px 宽度下每列约 200px，数据展示完整
- 平板端降级为 3 列，手机端 2 列

## 3. 禁止事项

| ❌ 禁止 | ✅ 替代方案 |
|--------|-----------|
| 使用 `any` 类型 | 定义明确的 TypeScript 接口 |
| 在组件中硬编码颜色 | 使用 CSS 变量 `--q-*` |
| 使用 `ref()` 存放大对象 | 使用 `shallowRef()` + `triggerRef()` |
| 状态变更后立即访问 DOM | `await nextTick()` 后再操作 |
| 复制工具函数到组件 | 统一放在 `utils/format.ts` |
| 使用纯白色 `#ffffff` | 使用 `#e6edf3` |
| 正收益只显示颜色不显示符号 | 同时显示 `+` 和绿色 |
