# Roadmap Research Summary — 2026-03-23

## What I Found

### Already Well-Implemented ✅
- **前复权 (forward-adjusted prices)**: Using stk_factor_pro API — correct
- **Commission + slippage**: 0.03% / 0.01% — implemented correctly
- **Volatility-based position sizing**: Target volatility approach with regime awareness — solid
- **ATR-based stop losses**: Bull/bear/sideways multipliers — good
- **Trading calendar (partial)**: Derived from OHLCV data — works but fragile

### Critical Gaps (Must Fix) 🔴

1. **T+1 Settlement (A股) — NOT IMPLEMENTED**
   - A股 requires shares bought today cannot be sold today
   - Current tracker has NO buy-date tracking per position
   - Impact: Return overestimation of ~5-15%
   - Fix: Add `buyDate` to Position struct, enforce settlement rule before sells

2. **Trading Calendar — Fragile**
   - Currently: SELECT DISTINCT trade_date FROM ohlcv_daily_qfq
   - This only works if OHLCV data is complete for all trading days
   - Risk: Miss a holiday sync gap → wrong calendar → signal misalignment
   - Fix: Use tushare `trade_cal` API with explicit `is_open` flag

3. **Short Selling Cost = 0**
   - A股 margin interest ~10.6% annual, but no modeling in tracker
   - Also: no borrow cost for hard-to-borrow small caps
   - Fix: Add daily margin interest accrual on short position value

### Important Gaps (Should Add) 🟡

4. **Stock Screener** — no dynamic filtering by fundamentals
5. **Factor Scoring System** — domain types exist but no scoring engine
6. **Market Impact** — flat slippage assumption is wrong for illiquid stocks
7. **VaR/CVaR** — domain types exist but no calculation
