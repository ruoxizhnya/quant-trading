# 量化交易系统 - 产品功能规划

_龙少（CEO）出品 · 2026-03-22_

---

## 一、愿景

做一个 **专业但不复杂** 的量化交易工具：
- 目标用户：散户、有一定交易经验的个人投资者
- 核心价值：把机构级的多因子策略和动态风控，带给普通投资者
- 差异化：配置驱动、热插拔策略、动态风控、A股优先但市场无关架构

---

## 二、产品路线图（V1.0 → V2.0）

### Phase 1：回测系统（V1.0）— 现在开始
**目标：跑通"选股 → 回测 → 报告"完整闭环**

#### 用户故事
1. 用户选择/配置一个策略（YAML）
2. 用户选择股票池（单只、一组、指数成分）
3. 用户选择时间范围
4. 系统输出回测报告：收益率、夏普率、最大回撤、交易明细

#### 功能列表

**Data Service**
- [ ] tushare.pro 数据拉取（日线 OHLCV、财务数据、指数成分）
- [ ] TimescaleDB 存储（时序优化）
- [ ] 数据校验和清洗
- [ ] 本地 Redis 缓存层

**Strategy Service**
- [ ] 策略热插拔接口（Strategy Interface）
- [ ] 策略注册表（Strategy Registry）
- [ ] 第一个策略：value_momentum（PE + PB + 动量 + ROE）
- [ ] YAML 配置驱动（不改代码，只改配置）

**Risk Service**
- [ ] 波动率目标仓（Volatility Targeting）
- [ ] 市场状态检测（bull/bear/sideways）
- [ ] 动态止损（ATR 倍数，随市场状态浮动）

**Analysis Service（回测引擎）**
- [ ] 时间序列遍历（日级别）
- [ ] 每日信号生成
- [ ] 模拟成交（日收盘价）
- [ ] 组合净值计算
- [ ] 报告生成（收益率、夏普率、最大回撤、Calmar 比率、胜率）

**共享基础设施**
- [ ] 微服务 HTTP API（Gin 框架）
- [ ] 结构化日志（zerolog）
- [ ] YAML 配置管理（viper）
- [ ] Docker Compose 本地开发环境

---

### Phase 2：实时信号系统（V1.5）
**目标：从回测到实时信号**

- [ ] 定时任务（每日收盘后自动拉数据）
- [ ] 实时信号生成（当前持仓建议）
- [ ] 邮件/推送通知（可选）

---

### Phase 3：模拟实盘（V2.0）
**目标：模拟真实交易环境**

- [ ] Execution Service 完整实现（模拟下单）
- [ ] 持仓管理（买入/卖出/持仓跟踪）
- [ ] 动态调仓（月度/季度再平衡）
- [ ] 绩效归因分析

## 三、技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 语言 | Go 1.25+ | 高并发、轻量、微服务友好 |
| 数据库 | PostgreSQL + TimescaleDB | 时序数据支持、SQL 生态 |
| 缓存 | Redis | 数据缓存、会话存储 |
| HTTP 框架 | Gin | 轻量、成熟、文档好 |
| 配置管理 | Viper | YAML 支持、环境变量 |
| 日志 | zerolog | 结构化、性能好 |
| 容器化 | Docker Compose | 本地开发 |
| 部署（未来） | Kubernetes | 生产环境 |

## 五、第一个里程碑（72小时内可交付）

**MVP 功能集（最小可行产品）：**

1. Data Service 能从 tushare 拉数据并存 TimescaleDB
2. value_momentum 策略实现（可配置 YAML）
3. 回测引擎能跑完完整回测并输出报告
4. Docker Compose 一键启动（DB + Redis + 5个 Service）

**验收标准：**
- 拉取 50 只股票 1 年日线数据
- 用 value_momentum 策略跑 2024 年全年回测
- 输出：总收益率、夏普率、最大回撤、交易次数

## 七、风险与边界

**已知限制：**
- tushare.pro 免费 token 有频率限制（200请求/分钟）
- 回测用收盘价模拟，不代表真实成交
- 不做任何投资建议，系统仅供参考

**不做的事：**
- 不接入真实券商 API（合规风险）
- 不做高频交易（基础设施不够）
- 不保证收益（市场有风险）
